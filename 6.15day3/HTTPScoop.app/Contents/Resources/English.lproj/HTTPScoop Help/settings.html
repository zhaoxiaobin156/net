Control what's captured with Preferences

The Preferences screen (accessed from the HTTPScoop menu when a capture is not in progress) allows you to describe what traffic you would like to capture. These settings are recorded in your user preferences, so be aware that they will be reloaded automatically next time you run the application.

Settings:

  • Promiscuous Mode
  • Restrict to Hosts
  • Restrict to Ports

Promiscuous Mode

Promiscuous mode allows you to capture HTTP conversations that do not involve the machine you are running HTTP Scoop on. This can be useful when neither server nor client are running on OS X. It can also be handy in production or integration testing environments where you need to examine a server’s behaviour to various clients in-situ.

Note that in order for HTTP Scoop to be able to capture HTTP conversations, the traffic must reach your computer’s network adapter. Generally on a switched network you will only be able to see traffic where your machine is the client or the server. A solution to this is to plug your computer into a network hub to which the server or client to be monitored is also connected. Hubs are inexpensive devices which pass all traffic to and from any connected port to all ports on the hub. Certain network switches can be configured to emulate a hub to achieve the same behaviour.

You can also listen in on conversations conducted over an unsecured wireless network although this may not work with all wireless routers. Please contact support@tuffcode.com to report any configurations which you find do not work.

Finally, a word of warning: the use of promiscuous mode can obviously lead to an invasion of privacy. Please be aware that putting your network into adapter into promiscuous mode can be detected and is often frowned upon. Always seek authorisation from the network’s administrators before using this facility.

Restrict to Hosts

This option allows you to limit the conversations which are captured to those where the client or server appears in the list. As well as reducing clutter, this facility provides one way of avoiding the "Traffic too fast: some packets lost" message (see Solving Problems) by allowing the OS to quickly disregard packets which you are not interested in.

You may specify host names e.g. www.tuffcode.com as well as IP addresses. Be aware that this can have some unexpected side-effects however: see the Requests missing from capture troubleshooting topic for more information on this.

Restrict to Ports

By default, HTTP Scoop will only look for HTTP conversations conducted over port 80. To capture on other TCP ports, you can specify them here. Don’t forget to explicitly add port 80 to the list if you want to capture other ports in addition to the default.
